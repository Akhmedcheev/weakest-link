# Система аналитики раунда - "Слабое звено"

Один из главных документов (список всех — в [RULES.md](RULES.md)).

## Обзор
Создана продвинутая система аналитики для оператора, которая автоматически открывается при переходе в состояние `Voting` и предоставляет детальную статистику по раунду.

## Компоненты

### 1. StatsAnalyzer.cs
**Расположение:** `Core/Analytics/StatsAnalyzer.cs`

**Основные классы:**
- `StatsAnalyzer` - главный класс анализа
- `PlayerRoundStats` - статистика игрока за раунд
- `RoundAnalytics` - полная аналитика раунда

**Функциональность:**
- Сбор статистики по игрокам (верные/неверные ответы, пасы, деньги в банк)
- Расчет процента успешности
- Определение сильного/слабого звена по строгой математике
- Прогноз выбывания на основе комплексного анализа
- Расчет эффективности раунда

### 2. RoundStatsWindow.xaml
**Расположение:** `Views/RoundStatsWindow.xaml`

**UI элементы:**
- **Прогноз выбывания** - крупный красный текст с предупреждением
- **Таблица игроков** - DataGrid с цветовой индикацией:
  - Зеленый - верные ответы
  - Красный - неверные ответы  
  - Оранжевый - пасы
  - Голубой - деньги в банк
  - Золотой - процент успешности
- **Статистика раунда** - общие показатели:
  - Собрано/сгорело денег
  - Длительность и среднее время ответа
  - Сильное/слабое звено
  - Общая эффективность

### 3. Интеграция в OperatorPanel
**Автоматическое открытие:** при переходе в `GameState.Voting`
**Ручное открытие:** кнопка "ANALYTICS 📊"
**Автоматическое закрытие:** при выходе из `Voting`

## Алгоритмы анализа

### Сильное звено
1. Максимум верных ответов
2. При равенстве - максимум денег в банк
3. При равенстве - максимальный процент успешности

### Слабое звено
1. Максимум неверных ответов
2. При равенстве - минимум денег в банк
3. При равенстве - минимальный процент успешности

### Прогноз выбывания
Комплексная система голосования:
- Слабое звено (1 голос)
- Игрок с максимальными потерями (1 голос)
- Худший процент успешности (1 голос)
- Минимум верных ответов (1 голос)

Победитель определяется по большинству голосов. При равенстве приоритет у слабого звена.

## Цветовая индикация в таблице
- **Зеленая подсветка** - сильное звено
- **Красная подсветка** - слабое звено  
- **Оранжевая рамка** - прогноз выбывания (если не совпадает со слабым звеном)

## Особенности реализации
- Thread-safe обновление через Dispatcher.Invoke
- Автоматическое обновление данных при изменениях
- Обработка ошибок с логированием
- Модальное окно с owner = OperatorPanel
- Автоматическая очистка при закрытии программы

## Использование
1. **Автоматически:** окно открывается при голосовании
2. **Вручную:** кнопка "ANALYTICS 📊" в панели управления
3. **Обновление:** данные обновляются в реальном времени
4. **Закрытие:** автоматически при выходе из голосования или вручную

## Расширяемость
Система готова для добавления:
- Графиков и диаграмм
- Истории предыдущих раундов
- Экспорта статистики
- Сравнительного анализа игроков
