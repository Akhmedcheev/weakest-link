# Исправленная математика аналитики - "Слабое звено"

Один из главных документов (список всех — в [RULES.md](RULES.md)).

## Проблема
Исходная логика некорректно обрабатывала "Пасы" - они не учитывались как отрицательный фактор при определении слабого звена.

## Исправления

### 1. Обновлен класс `PlayerRoundStats`

#### Новые свойства:
```csharp
// Суммарные ошибки (неверные + пасы)
public int TotalMistakes => IncorrectAnswers + Passes;

// Общий процент успешности с учетом пасов как ошибок
public double SuccessPercentage => TotalQuestions > 0 ? 
    Math.Round((double)CorrectAnswers / TotalQuestions * 100, 1) : 0;
```

### 2. Новая математика определения "Слабого звена"

#### Старая логика:
1. Максимум неверных ответов
2. Минимум денег в банк  
3. Худший процент успешности

#### Новая логика:
1. **Максимум суммарных ошибок** (Неверно + Пасы)
2. **При равенстве ошибок** - минимум денег в банк
3. **При равенстве денег** - минимум верных ответов
4. **При равенстве верных** - худший процент успешности

```csharp
return playerStats
    .OrderByDescending(p => p.TotalMistakes)  // Ошибки + Пасы
    .ThenBy(p => p.BankedMoney)               // Меньше денег в банк
    .ThenBy(p => p.CorrectAnswers)            // Меньше верных ответов
    .ThenBy(p => p.SuccessPercentage)         // Худший процент
    .First().Name;
```

### 3. Улучшенный прогноз выбывания

#### Новые факторы с весами:
- **Слабое звено** - вес 3 (самый важный)
- **Максимум ошибок** - вес 2 (важный)
- **Потери денег** - вес 1 (средний)
- **Максимум пасов** - вес 1 (средний)
- **Худший процент** - вес 1 (средний)
- **Минимум верных** - вес 1 (дополнительный)

#### Дополнительные кандидаты:
- Игрок с максимальным количеством пасов
- Игрок с максимальным общим количеством ошибок

### 4. Обновлен UI

#### Новый столбец "Ошибки":
- Показывает сумму (Неверно + Пасы)
- Выделен красным жирным шрифтом
- Помогает визуально определить слабое звено

#### Улучшенная сортировка таблицы:
1. Процент успешности (по убыванию)
2. Количество ошибок (по возрастанию)
3. Деньги в банк (по убыванию)

## Пример работы

### Сценарий:
- **Игрок А**: 2 верно, 0 неверно, 2 паса, 0 ₽ в банк
- **Игрок Б**: 2 верно, 2 неверно, 0 пасов, 5000 ₽ в банк

### Старая логика:
- Слабое звено: Игрок Б (2 неверных против 0)

### Новая логика:
- Игрок А: 2 ошибки (0 неверно + 2 паса)
- Игрок Б: 2 ошибки (2 неверно + 0 пасов)
- При равенстве ошибок: Игрок А слабее (0 ₽ против 5000 ₽ в банк)
- **Слабое звено: Игрок А** ✅

## Преимущества новой системы

1. **Справедливость**: Пасы теперь правильно наказываются
2. **Математическая корректность**: Учитываются все негативные факторы
3. **Визуальная ясность**: Новый столбец "Ошибки" наглядно показывает проблемных игроков
4. **Точный прогноз**: Многокритериальная система с весами дает более точные предсказания
5. **Логичная сортировка**: Таблица отсортирована по реальной эффективности

## Влияние на игру

- Игроки не смогут злоупотреблять пасами
- Слабое звено определяется более объективно
- Прогноз выбывания становится более точным
- Статистика лучше отражает реальный вклад игроков

Система теперь полностью соответствует правилам игры "Слабое звено", где пасы считаются таким же негативным фактором, как и неверные ответы.
