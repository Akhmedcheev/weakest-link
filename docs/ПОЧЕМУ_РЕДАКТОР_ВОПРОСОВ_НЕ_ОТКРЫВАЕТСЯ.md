# Почему редактор вопросов не открывается

В этом документе описаны возможные причины, по которым редактор вопросов может не запускаться при нажатии кнопки «✎ РЕДАКТОР ВОПРОСОВ» или пункта меню «ОТКРЫТЬ РЕДАКТОР ВОПРОСОВ».

---

## Как работает открытие редактора

При нажатии на кнопку выполняется следующая логика:

1. **Поиск отдельного exe**  
   Ищется файл `WeakestLink.QuestionEditor.exe`:
   - сначала в папке запуска основного приложения (`bin\Debug\net10.0-windows\` или `bin\Release\...`);
   - если не найден — по альтернативному пути:  
     `QuestionEditor\bin\Debug\net10.0-windows\WeakestLink.QuestionEditor.exe`  
     относительно папки вывода основного приложения.

2. **Запуск**  
   - если exe найден — запускается отдельный процесс;
   - если не найден — открывается встроенное окно `QuestionEditorWindow` внутри основного приложения.

---

## Возможные причины и решения

### 1. Проект QuestionEditor не собран

**Симптом:** exe не найден, вместо него открывается встроенное окно, либо падает ошибка.

**Причина:** В `WeakestLink.csproj` сборка QuestionEditor выполняется с `IgnoreExitCode="true"`, поэтому неуспешная сборка не останавливает основной проект, и exe может не появиться.

**Решение:**
- Выполните явную сборку:  
  `dotnet build QuestionEditor\WeakestLink.QuestionEditor.csproj -c Debug`
- Убедитесь, что в `QuestionEditor\bin\Debug\net10.0-windows\` появился `WeakestLink.QuestionEditor.exe`.

---

### 2. Exe не копируется в папку основного приложения

**Симптом:** exe есть в `QuestionEditor\bin\...`, но редактор всё равно не запускается или открывается встроенное окно.

**Причина:** Копирование exe в `$(OutDir)` выполняется только при успешной сборке QuestionEditor. Если exe не был собран, его нет и в выходной папке основного проекта.

**Решение:**
- Откройте основное приложение из Visual Studio или `dotnet run`.
- Проверьте наличие `WeakestLink.QuestionEditor.exe` рядом с `WeakestLink.exe` в папке вывода (например, `bin\Debug\net10.0-windows\`).

---

### 3. Антивирус блокирует запуск

**Симптом:** exe найден, но при запуске ничего не происходит или появляется сообщение о блокировке.

**Решение:**
- Добавьте папку с проектом и exe в исключения антивируса.
- Если редактор не нужен как отдельный exe — используйте встроенное окно (запустится при отсутствии exe).

---

### 4. Ошибка при запуске встроенного окна (QuestionEditorWindow)

**Симптом:** exe не найден, должно открыться встроенное окно, но приложение падает или окно не появляется.

**Причина:** `QuestionEditorWindow` может выбросить исключение при инициализации (например, из-за XAML, отсутствия `questions.json` или несовместимой структуры JSON).

**Решение:**
- Проверьте, что в рабочей папке приложения есть `questions.json` в ожидаемом формате.
- Запустите приложение под отладчиком и посмотрите стек исключения в момент нажатия на кнопку редактора.

---

### 5. Падение отдельного приложения QuestionEditor при старте

**Симптом:** exe запускается, окно не появляется, процесс может сразу завершиться.

**Причина:** В `QuestionEditor\App.xaml.cs` настроена обработка необработанных исключений. При ошибке создаётся файл `question_editor_crash.txt` в текущей рабочей папке и показывается MessageBox.

**Решение:**
- Посмотрите `question_editor_crash.txt` в папке запуска редактора (обычно `bin\Debug\net10.0-windows\`).
- Часто это ошибка сериализации/десериализации JSON при загрузке `questions.json` или `final_questions.json`.

---

### 6. Некорректный путь (пробелы, спецсимволы)

**Симптом:** Ошибка вида «файл не найден» или «путь недопустим».

**Причина:** Путь проекта содержит пробелы (`WEAKEST LINK SOPFTWARE`), что в редких случаях может влиять на запуск процессов.

**Решение:**
- Проверьте, что путь к exe не содержит недопустимых символов.
- При необходимости запускайте приложение из папки без пробелов в пути.

---

### 7. Отсутствует .NET 10 (net10.0-windows)

**Симптом:** exe найден, но при запуске появляется сообщение о том, что не найдена среда выполнения.

**Причина:** Проект собран под `net10.0-windows`. Без установленного .NET 10 (или совместимой версии) приложение не запустится.

**Решение:**
- Установите нужный workload/Runtime для .NET 10 (Desktop/WPF).

---

## Краткая диагностика

| Симптом | Проверить |
|--------|-----------|
| Ничего не происходит | Есть ли exe рядом с основным приложением? Антивирус? |
| «Не удалось запустить редактор: ...» | Точный текст ошибки, `question_editor_crash.txt` |
| Окно закрывается сразу | `question_editor_crash.txt`, формат JSON в `questions.json` |
| Открывается встроенное окно | Ожидаемо, если exe не найден |

## Где смотреть в коде

- Запуск редактора: `Views\OperatorPanel.xaml.cs` → метод `BtnOpenQuestionEditor_Click`

- Отдельное приложение: папка `QuestionEditor\`
- Сборка и копирование exe: `WeakestLink.csproj` → таргет `BuildQuestionEditor`
