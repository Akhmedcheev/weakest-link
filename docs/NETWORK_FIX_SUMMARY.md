# Исправления сетевой логики проекта "Слабое звено"

Один из главных документов (список всех — в [RULES.md](RULES.md)).

## Проблема
При запуске серверов приложение часто захватывало IP-адреса от VPN (Hamachi, Radmin, OpenVPN) или виртуальных адаптеров, что мешало подключению локальных устройств (iPad).

## Решение

### 1. Улучшен метод `GetLocalIPAddress()` в `OperatorPanel.xaml.cs`

#### Добавлена жесткая фильтрация сетевых интерфейсов:
- Только `Ethernet` и `Wireless80211` со статусом `OperationalStatus.Up`
- Расширенный список исключений VPN/виртуальных адаптеров:
  - "virtual", "vpn", "hamachi", "hyper-v", "vmware", "tunnel", "tailscale"
  - "radmin", "pseudo", "loopback", "zerotier", "openvpn", "wireguard"
  - "tap", "tun", "veth", "docker", "bridge", "kvm", "qemu", "virtualbox"

#### Дополнительные проверки:
- Исключение интерфейсов без MAC-адреса
- Исключение loopback (127.0.0.1) и APIPA (169.254.*.*) адресов
- Валидация финального IP через метод `IsValidLocalIp()`

### 2. Добавлено подробное логирование
- Отображение всех найденных интерфейсов и причин их исключения
- Показ выбранного IP-адреса и интерфейса
- Итоговая сводка всех сетевых настроек

### 3. Улучшен `WebRemoteController.cs`
- Добавлено логирование всех привязанных адресов при старте
- Улучшена обработка ошибок

## Результат
Теперь приложение гарантированно выберет "чистый" IP-адрес от физического сетевого адаптера (Ethernet или Wi-Fi), игнорируя все виртуальные и VPN подключения.

## Использование
- TCP сервер: порт 8888 (слушает на всех интерфейсах через IPAddress.Any)
- Web Remote: порт 8080 (или 8080→8081 если занят)
- QR-код и ссылка генерируются с отфильтрованным IP-адресом

## Диагностика
При запуске приложения в логе появится подробная информация о выборе IP-адреса, что поможет в случае проблем с подключением.
