# Система подготовки нового раунда - "Слабое звено"

Один из главных документов проекта (полный список — в [RULES.md](RULES.md)).

## Обзор
Создана полноценная система подготовки к новому раунду, которая автоматически выполняет все необходимые действия при нажатии кнопки "NEXT ROUND".

## Компоненты

### 1. GameEngine.cs - Новые методы

#### `PrepareNewRound()`
Основной метод подготовки нового раунда:
- Автоматический переход к следующему раунду
- Очистка статистики раунда (банк, цепочка)
- Определение стартового игрока по алгоритму сильного звена
- Инициализация статистики для всех активных игроков
- Уведомление UI об изменениях

#### `GetNewRoundDuration()`
Возвращает время для нового раунда (вызывает `GetRoundDuration()`). Уменьшение на 10 секунд от раунда к раунду:
- Раунд 1: 150 сек (2:30)
- Раунд 2: 140 сек (2:20)
- Раунд 3: 130 сек (2:10)
- Раунд 4: 120 сек (2:00)
- Раунд 5: 110 сек (1:50)
- Раунд 6: 100 сек (1:40)
- Раунд 7 и далее: 90 сек (1:30) при двух игроках (предфинал), иначе 150 сек

### 2. OperatorPanel.xaml.cs - Обновленная логика

#### `BtnNextRound_Click()` - Полностью переработан
Выполняет последовательную подготовку раунда:

1. **Очистка экрана** от предыдущего раунда
2. **Подготовка движка** через `_engine.PrepareNewRound()`
3. **Сброс таймера** на время нового раунда
4. **Отправка времени** на все экраны
5. **Определение стартового игрока** и рассылка на экраны
6. **Загрузка вопроса** для нового раунда
7. **Переход в состояние** `RoundReady`
8. **Детальное логирование** процесса

#### `UpdateStateButtons()` - Исправлена логика
Кнопка **PLAY** теперь активна в состояниях:
- `RoundReady` - для запуска таймера
- `Playing` - во время игры

### 3. HostScreen.xaml.cs - Новые возможности

#### `UpdateCurrentPlayer(string playerName)`
Отображает текущего игрока на экране ведущего:
- Показывает "ХОД ИГРОКА: [ИМЯ]" желтым цветом
- Автоматически восстанавливает стандартный заголовок при необходимости

#### Обработка команды `CURRENT_PLAYER`
Принимает и обрабатывает информацию о текущем игроке от OperatorPanel.

## Алгоритм определения стартового игрока

### Раунд 1:
- Первый игрок по алфавиту

### Раунды 2+:
1. **Сильное звено прошлого раунда** (максимум верных ответов)
2. **При равенстве** - максимум денег в банк
3. **Если сильное звено выбыло** - второй по силе игрок
4. **Если данных нет** - сильное звено по всем раундам
5. **Иначе** - первый по алфавиту

## Процесс подготовки раунда

### Автоматические действия:
1. ✅ Очистка вопроса и ответа
2. ✅ Инкремент номера раунда
3. ✅ Сброс банка и цепочки
4. ✅ Определение стартового игрока
5. ✅ Установка таймера (короче на 10 сек)
6. ✅ Загрузка первого вопроса
7. ✅ Переход в `RoundReady`
8. ✅ Рассылка данных на все экраны

### Сетевые команды:
- `CLEAR_ELIMINATION` - очистка экрана
- `UPDATE_TIMER|{seconds}` - обновление таймера
- `CURRENT_PLAYER|{name}` - текущий игрок
- `SET_STATE|RoundReady` - состояние готовности

## Логирование
Система ведет подробный лог процесса подготовки:
```
=== ПОДГОТОВКА НОВОГО РАУНДА ===
Стартовый игрок раунда 2: Иванов
Загружен вопрос для раунда 2: Вопрос...
Раунд 2 подготовлен:
  - Время: 140 сек (2:20)
  - Стартовый игрок: Иванов
  - Активных игроков: 8
========================
```

## Взаимодействие с UI
- Кнопка **PLAY** активируется только после `RoundReady`
- Отображение текущего игрока на HostScreen
- Синхронизация таймера на всех экранах
- Автоматическая загрузка вопроса на пульт оператора

## Преимущества новой системы
- **Полная автоматизация** - одно нажатие готовит весь раунд
- **Интеллектуальный старт** - сильное звено начинает раунд
- **Правильное время** - автоматическое уменьшение на 10 секунд
- **Синхронизация** - все экраны получают актуальные данные
- **Отказоустойчивость** - детальная обработка ошибок
- **Прозрачность** - подробное логирование процесса

Система готова к использованию и полностью интегрирована в игровой процесс!
