# Статистика голосования и механизм аналитики — «Слабое звено»

Документ описывает всё, что сделано по сбору статистики раунда, аналитике и механизму голосования (состояние Voting, окно аналитики, исключение игрока).

---

## 1. Общая схема

1. **Во время раунда (Playing)** оператор нажимает «Верно» / «Неверно» / «Пас» / «Банк» — движок записывает статистику по текущему игроку за текущий раунд.
2. **По окончании времени** (или при достижении полного банка 50 000) игра переходит в состояние **Voting**.
3. **В Voting** автоматически открывается окно аналитики раунда с таблицей игроков, прогнозом выбывания, сильным/слабым звеном и т.д.
4. Оператор смотрит аналитику, при необходимости выбирает игрока в выпадающем списке и нажимает **«ИСКЛЮЧИТЬ»** — игрок удаляется, игра переходит в **Elimination**, затем в **RoundReady** или **FinalDuel**.
5. При выходе из Voting окно аналитики закрывается автоматически.

**Исключение:** в раунде с двумя игроками (предфинал, 7-й раунд) голосования нет — раунд завершается сразу в Idle, банк удваивается и добавляется к общей сумме, показывается кнопка «ПЕРЕЙТИ К ФИНАЛУ».

---

## 2. Состояние игры (State Machine) и Voting

- **Файл:** `Core/GameState.cs`
- Состояния: `Idle`, `RoundReady`, `Playing`, **`Voting`**, `Elimination`, `FinalDuel`.
- **Voting** — «Раунд окончен, таймер остановлен, идёт голосование».

Переходы:
- **Playing → Voting:** по истечении таймера раунда (или при досрочном достижении полного банка) — кроме случая с 2 игроками.
- **Voting → Elimination:** после нажатия «ИСКЛЮЧИТЬ» в панели оператора (вызов `_engine.EliminatePlayer(targetName)`).
- **Voting → Idle:** при ручном переходе по кнопке состояния (редко).

Логика переходов и проверки допустимости — в `Core/GameEngine.cs`, метод `TransitionTo(GameState newState)`.

## 3. Сбор статистики во время раунда

### 3.1. Хранение данных

- **Файл:** `Core/GameEngine.cs`
- Структура: `Dictionary<string, Dictionary<int, PlayerStats>> PlayerStatistics`
  - Внешний ключ — имя игрока.
  - Внутренний ключ — номер раунда (`CurrentRound`).
  - Значение — объект `PlayerStats`.

Класс **PlayerStats** (в том же файле):

| Поле | Описание |
|------|----------|
| `CorrectAnswers` | Количество верных ответов |
| `IncorrectAnswers` | Количество неверных ответов |
| `Passes` | Количество пасов |
| `BankedMoney` | Сумма денег, положенных в банк в этом раунде |
| `BankPressCount` | Количество нажатий «Банк» (сколько раз игрок клал деньги в банк) |
| `BurnedByWrongAnswers` | Сумма, сгоревшая из-за неверных ответов этого игрока |
| `ExactDroppedMoney` | Точная сумма чужих денег, сброшенных при неверном ответе или пасе |

### 3.2. Когда записывается статистика

- **Верный ответ:** `CorrectAnswer()` → `UpdatePlayerStat(CurrentPlayerTurn, s => s.CorrectAnswers++)`, сдвиг по цепочке, переход хода.
- **Неверный ответ:** `WrongAnswer()` → `UpdatePlayerStat(CurrentPlayerTurn, s => s.IncorrectAnswers++)`, сброс цепочки, переход хода.
- **Пас:** `Pass()` → `UpdatePlayerStat(CurrentPlayerTurn, s => s.Passes++)`, сброс цепочки, переход хода.
- **Банк:** `Bank()` → `UpdatePlayerStat(CurrentPlayerTurn, s => { s.BankedMoney += valueToBank; s.BankPressCount++; })`, начисление в RoundBank, сброс цепочки.

Инициализация записей по раундам выполняется при переходе в новый раунд (подготовка раунда в движке): для каждого `ActivePlayers` создаётся запись `PlayerStatistics[playerName][CurrentRound]` при необходимости.

## 4. Анализатор статистики (StatsAnalyzer)

- **Файл:** `Core/Analytics/StatsAnalyzer.cs`
- Используется для расчёта аналитики **по текущему раунду** на момент перехода в Voting.

### 4.1. Классы

- **PlayerRoundStats** — статистика одного игрока за раунд (для отображения и расчётов):
  - Те же поля, что в `PlayerStats`, плюс вычисляемые: `TotalMistakes` (неверные + пасы), `AverageBankAmount` (BankedMoney / BankPressCount при BankPressCount > 0), `SuccessPercentage` (CorrectAnswers / TotalQuestions * 100), `ChainBreaksLost` (оценочные потери при обрывах цепочки).
- **RoundAnalytics** — результат анализа раунда:
  - Список `PlayerRoundStats`, суммы по раунду (TotalBanked, TotalBurned), длительность, количество вопросов, среднее время ответа, **StrongestLink**, **WeakestLink**, **EliminationPrediction**, **EliminationReason**, **PanicBanker**, эффективность раунда (EfficiencyPercentage) и т.д.

### 4.2. Алгоритмы

- **Сильное звено (DetermineStrongestLink):** максимум верных ответов → при равенстве максимум денег в банк → при равенстве максимум процента успешности.
- **Слабое звено (DetermineWeakestLink):** максимум суммарных ошибок (неверные + пасы) → при равенстве минимум денег в банк → при равенстве минимум верных ответов → при равенстве минимум процента успешности.
- **Главный перестраховщик / Panic Banker (DeterminePanicBanker):** среди игроков с `BankPressCount > 0` выбирается тот, у кого **минимальный средний размер банка** (BankedMoney / BankPressCount) — то есть кто чаще всего «мелко» банковал.
- **Прогноз выбывания (PredictElimination, PredictFromCandidates):**
  - Игроки с верными ответами и без ошибок (`CorrectAnswers > 0 && TotalMistakes == 0`) считаются защищёнными и по умолчанию не кандидатами.
  - Если все защищены — кандидат на выбывание: с наименьшим числом верных ответов.
  - Иначе среди незащищённых применяется «голосование» по нескольким факторам с весами:
    - Слабое звено — вес 2
    - Максимум ошибок (TotalMistakes) — вес 2
    - Максимум потерь при обрывах (ChainBreaksLost) — вес 1
    - Максимум пасов — вес 1
    - Худший процент успешности — вес 1
    - Минимум верных ответов — вес 1
  - Победитель — кандидат с максимальной суммой «голосов»; при равенстве приоритет у слабого звена. В `EliminationReason` записывается текстовое объяснение (какие критерии «проголосовали» за этого игрока).

Подробности и примеры — в `ELIMINATION_PREDICTION_FIX.md` и `PANIC_BANKER_METRIC.md`.

## 5. Окно аналитики раунда (RoundStatsWindow)

- **Файлы:** `Views/RoundStatsWindow.xaml`, `Views/RoundStatsWindow.xaml.cs`
- Отображает результат `StatsAnalyzer.AnalyzeRound(roundDurationSeconds)`.

### 5.1. Содержимое окна

- **Прогноз на выбывание** — крупный красный текст с именем кандидата и (при наличии) объяснением `EliminationReason`.
- **Таблица игроков (DataGrid):** имя, верно, неверно, пасы, ошибки (TotalMistakes), в банк, средний банк, успешность (%). Сортировка по умолчанию: по успешности (убыв.), по ошибкам (возр.), по деньгам в банк (убыв.).
- **Правая колонка — статистика раунда:** собрано в банк, сгорело, длительность раунда, всего вопросов, среднее время ответа, сильное звено, слабое звено, главный перестраховщик (с подписью среднего банка), эффективность раунда.

### 5.2. Цветовая подсветка строк таблицы

- **Зелёный фон** — сильное звено.
- **Красный фон** — слабое звено.
- **Фиолетовый фон** — главный перестраховщик (Panic Banker).
- **Оранжевая рамка** — прогноз выбывания, если он не совпадает со слабым звеном.

### 5.3. Открытие и закрытие

- **Автооткрытие:** при переходе движка в `GameState.Voting` из OperatorPanel вызывается `OpenRoundAnalytics()`: создаётся `RoundStatsWindow(_engine, roundDuration)`, задаётся Owner = OperatorPanel, окно показывается через `Show()` (не модально).
- **Автозакрытие:** при выходе из Voting (переход в Elimination или Idle) вызывается `CloseAnalytics()` — окно закрывается, ссылка обнуляется.
- **Вручную:** кнопка «ANALYTICS 📊» на панели оператора открывает то же окно аналитики (если уже открыто — предыдущее закрывается, создаётся новое с актуальными данными).
- Длительность раунда для аналитики: `_engine.GetRoundDuration() - _timeLeftSeconds` (фактическое время раунда в секундах).

Метод `RefreshAnalytics()` обновляет данные в уже открытом окне (пересчёт и перепривязка таблицы, текстов прогноза и статистики).

## 6. Панель оператора: голосование и исключение

- **Файлы:** `Views/OperatorPanel.xaml`, `Views/OperatorPanel.xaml.cs`

### 6.1. Кнопки голосования и блок Voting

- **VOTING START** (`BtnVotingStart_Click`) — старт голосования: трек `voting_dictor_part.mp3`, таймер 29 сек, строка состояния «ГОЛОСОВАНИЕ НАЧАЛОСЬ». С кроссфейдом в `general_bed.mp3` по окончании.
- **VOTING END** (`BtnVotingEnd_Click`) — конец голосования: трек `voting_host_part.mp3`, таймер 15 сек, строка «ГОЛОСОВАНИЕ ПОДХОДИТ К КОНЦУ».
- Обе кнопки доступны только в состоянии `Voting` и блокируются после нажатия до следующего раунда.
- Блок **VotingBorder** содержит:
  - Заголовок «ГОЛОСОВАНИЕ»
  - **ComboBox** `EliminationComboBox` — список игроков для исключения (`ItemsSource = _engine.ActivePlayers`, при инициализации панели; список общий с движком, поэтому при смене раунда отображаются актуальные активные игроки).
  - Кнопка **«ИСКЛЮЧИТЬ»** (`BtnEliminate_Click`) — берёт выбранное имя из ComboBox, удаляет игрока из `_engine.ActivePlayers`, вызывает `_engine.EliminatePlayer(targetName)`, рассылает по сети `ELIMINATE|{name}`, проигрывает звук ухода, логирует.
  - Кнопка «ФОН ПОСЛЕ ИНТЕРВЬЮ» — только звук.

### 6.2. Когда показывается блок Voting

- При переходе в **RoundReady** для раундов 1–6 блок Voting отображается (`VotingBorder.Visibility = Visible`).
- Для 7-го раунда (предфинал, 2 игрока) блок Voting скрывается (`VotingBorder.Visibility = Collapsed`), показывается кнопка «ПЕРЕЙТИ К ФИНАЛУ» после завершения раунда.

### 6.3. Переход в Voting по таймеру

- В обработчике таймера раунда (`RoundTimer_Tick`): когда время заканчивается (`_timeLeftSeconds <= 0`), если состояние `Playing` и игроков **не** 2 — вызывается `_engine.TransitionTo(GameState.Voting)`. Если игроков 2 — переход в Idle, сохранение и удвоение банка, сообщение о завершении префинала.

## 7. Аудио и сеть при голосовании

- **VOTING START:** трек `voting_dictor_part.mp3` (one-shot) с кроссфейдом в `general_bed.mp3` (нарастание в последние 2–3 сек).
- **VOTING END:** трек `voting_host_part.mp3` (bed, loop).
- При нажатии «ИСКЛЮЧИТЬ»: `updated_walk_of_shame_bed.mp3` с кроссфейдом в `general_bed.mp3`.
- По сети рассылается `ELIMINATE|{имя игрока}` для синхронизации клиентов и экрана ведущего.
