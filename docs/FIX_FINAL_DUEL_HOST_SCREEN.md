# Исправление: обновление финальной дуэли на экране ведущего (HostScreen)

## Проблема

При игре на пульте оператора (`OperatorPanel`) в режиме **FinalDuel** на экране ведущего (`HostScreen`) визуально ничего не обновлялось:

- Панель финальной дуэли появлялась, но **имена оставались заглушками** («ФИНАЛИСТ 1», «ФИНАЛИСТ 2») вместо реальных имён финалистов (например, АЛЕКСЕЙ, БОРИС).
- **10 кружков-индикаторов** ответов (по 5 на игрока) оставались серыми, хотя оператор нажимал «Верно» / «Неверно» и на пульте кружки корректно становились зелёными/красными.

Пульт при этом отправлял команды `DUEL_UPDATE` по TCP и в логе было видно отправку данных дуэли с именами и очками.

---

## Причины (что было не так)

1. **Перезапись данными STATE**  
   При переходе в финал пульт рассылает по TCP сообщения `STATE|FinalDuel` и `SET_STATE|FinalDuel`. В `HostScreen` при получении любого из них вызывался `ApplyGameState(FinalDuel)`, который **принудительно выставлял** имена «ФИНАЛИСТ 1» / «ФИНАЛИСТ 2» и заливал все кружки серым.  
   Если это сообщение приходило **после** прихода `DUEL_UPDATE` (из-за порядка доставки по сети), уже обновлённые имена и цвета затирались.

2. **Зависимость только от TCP в одном процессе**  
   Когда оба окна (пульт и экран ведущего) работают в одном процессе, доставка `DUEL_UPDATE` по TCP на localhost могла быть ненадёжной или не успевать. Экран ведущего не имел альтернативного канала обновления от пульта.

---

## Что исправлено

### 1. HostScreen.xaml.cs

- **ApplyGameState(GameState.FinalDuel)**  
  Убрано присвоение имён финалистов и заливки кружков. При переходе в состояние финала теперь меняются только:
  - видимость панелей (дерево, нижняя панель, панель дуэли);
  - заголовок «ФИНАЛЬНАЯ ДУЭЛЬ».  
  Имена и индикаторы задаются **только** из данных дуэли (см. ниже).

- **UpdateDuelDisplay**  
  - При любом приходе `DUEL_UPDATE` (или прямом вызове) включается отображение панели дуэли: видимость, скрытие нижней панели и дерева, заголовок. Так экран корректно показывается даже если сообщение `STATE` пришло позже или не пришло.
  - Имена обновляются только при непустых значениях, с `Trim()` и `ToUpper()`.
  - Строки очков парсятся через `Split` по запятой с `RemoveEmptyEntries`; пустые и `null` обрабатываются.
  - Вся логика выполняется внутри **Dispatcher.Invoke**, чтобы обновление UI шло только из UI-потока.

- **UpdateIndicators**  
  - Для каждого значения очка используется `Trim()`.
  - Цвета: `"1"` → зелёный (верно), `"0"` → красный (неверно), иначе → серый (#222 из XAML).

- **Публичный метод SetDuelDisplay(string p1, string p2, string s1, string s2)**  
  Добавлен для прямого обновления экрана ведущего из пульта (без TCP). Внутри вызывается тот же `UpdateDuelDisplay`.

- Ошибки парсинга дуэли логируются в `Debug.WriteLine` вместо показа сообщения пользователю, чтобы не прерывать показ.

### 2. OperatorPanel.xaml.cs

- **Прямая передача данных на HostScreen при каждой отправке DUEL_UPDATE**  
  После каждого `_server.Broadcast("DUEL_UPDATE|...")` добавлен вызов  
  `_hostScreen?.SetDuelDisplay(TxtFinalist1.Text, TxtFinalist2.Text, p1s, p2s)`  
  в местах:
  - переход в финал (начальная отправка);
  - нажатие «Старт дуэли»;
  - нажатие «Дуэль: Верно»;
  - нажатие «Дуэль: Неверно»;
  - метод `UpdateFinalCirclesUI()` (при каждом обновлении кружков на пульте).

- **При открытии экрана ведущего во время финала**  
  В обработчике кнопки открытия экрана ведущего (`BtnOpenHost_Click`): если текущее состояние — `FinalDuel`, сразу вызывается `_hostScreen.SetDuelDisplay(...)` с текущими именами финалистов и строками очков `p1s`, `p2s`, чтобы не показывать заглушки «ФИНАЛИСТ 1/2» и серые кружки.

---

## Итог

- **Единственный источник правды** для имён и кружков на экране ведущего — данные дуэли (из `DUEL_UPDATE` или из прямого вызова `SetDuelDisplay`). Сообщения `STATE`/`SET_STATE` больше не перезаписывают эти данные.
- **Два канала обновления:** TCP (для удалённых экранов) и прямой вызов `SetDuelDisplay` (для экрана ведущего в том же процессе), поэтому обновление не зависит от порядка доставки по сети и от возможных особенностей localhost-TCP.
- Все обновления UI на HostScreen выполняются в **Dispatcher.Invoke**.
- Кружки на экране ведущего отображают: **зелёный** — верно, **красный** — неверно, **серый** — ещё без ответа.

Файлы изменений: `Views/HostScreen.xaml.cs`, `Views/OperatorPanel.xaml.cs`.
