# «Слабое звено» — Программно-аппаратный комплекс для телеигры

**Версия:** 2.0
**Обновлено:** 07.03.2026
**Фреймворк:** .NET WPF (C# / XAML)
**Статус:** Production-ready

---

## Описание

Полнофункциональный WPF-комплекс для проведения телеигры «Слабое звено» (Weakest Link). Включает пульт оператора, два варианта экрана ведущего (Classic / Modern 2020), систему суфлёра, аналитику раундов и финальную дуэль.

---

## Возможности

### Полный игровой цикл
- **PRE-GAME:** 4 фазы вступления (Opening → Intro → Представление → Правила → Ready)
- **7 раундов:** Денежная цепочка (1 000 — 50 000 ₽), таймер (150 → 90 сек)
- **Голосование:** 45 сек, VOTE START → REVEAL → VERDICT → ELIMINATE
- **Tie-breaker:** Автоматическое определение ничьи с двумя сценариями
- **Финальная дуэль:** 5 пар вопросов, Sudden Death, досрочная победа
- **Удвоение банка:** Предфинальный (7-й) раунд — банк ×2

### Пульт оператора
- Кнопки быстрого ответа (ВЕРНО / НЕВЕРНО / ПАС / БАНК) с визуальным эффектом нажатия
- StatusBar «Штурман»: три сегмента (ВЫ СДЕЛАЛИ / СЕЙЧАС / ПОРА НАЖАТЬ)
- Аналитика раунда: Сильное/Слабое звено, Panic Banker, прогноз выбывания
- Debug-панель и бот-режим для тестирования

### Суфлёр ведущего
- TCP-команда `HOST_MESSAGE` отображает текст на экране ведущего
- Динамические подстановки (имя первого игрока, сумма банка, имя Сильного звена)
- Match-ball и Sudden Death уведомления в финальной дуэли

### Сеть
- TCP Server/Client (порт 8888)
- Мгновенная синхронизация таймера, банка, вопросов, состояний на всех экранах

### Аудио
- 20+ треков для всех фаз игры (NAudio)
- Кроссфейды, fade-in, автоматический выбор трека по длительности раунда

## Структура проекта

```
Core/
├── GameEngine.cs           — Ядро: 13 состояний, банк, таймер, статистика, дуэль
├── GameState.cs            — Перечисление состояний
├── GameServer.cs           — TCP-сервер
├── GameClient.cs           — TCP-клиент
├── QuestionProvider.cs     — Загрузка вопросов из JSON
└── Analytics/
    └── StatsAnalyzer.cs    — Аналитика, tie-detection, strongest/weakest

Views/
├── OperatorPanel.xaml/.cs  — Пульт оператора
├── HostScreen.xaml/.cs     — Классический экран ведущего
├── HostScreenModern.xaml/.cs — Современный экран (2020)
└── RoundStatsWindow.xaml/.cs — Окно аналитики раунда

Audio/
├── AudioManager.cs         — NAudio: Play, Stop, Crossfade
└── Assets/Audio/           — Аудиофайлы (.mp3)

Data/
├── questions.json          — Вопросы для раундов
└── final_questions.json    — Вопросы для финальной дуэли
```

## Быстрый старт

1. Откройте решение в Visual Studio
2. Соберите проект (`dotnet build`)
3. Запустите OperatorPanel (главное окно)
4. Подключите экраны ведущего по TCP (порт 8888)
5. Добавьте участников → запустите игровую сессию
