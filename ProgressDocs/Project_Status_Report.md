# Статус проекта: «Слабое звено» (Weakest Link)

**Обновлено:** 07.03.2026
**Статус:** PRODUCTION READY

---

## Реализованная архитектура

### 1. Игровое ядро (Core/GameEngine.cs)

* **Денежная цепочка:** Полная математика (1 000 — 50 000 ₽). `RoundBank`, `TotalBank`, автоматический сброс при ошибке.
* **State Machine:** 13 состояний — `Idle`, `IntroOpening`, `IntroNarrative`, `PlayerIntro`, `RulesExplanation`, `RoundReady`, `Playing`, `RoundSummary`, `Voting`, `Discussion`, `Reveal`, `Elimination`, `FinalDuel`.
* **Учёт статистики:** Верные/неверные, пасы, банк, BurnedByWrongAnswers, ExactDroppedMoney.
* **Таймер:** Динамическое время (150 → 90 сек) с убыванием в каждом раунде.
* **Удвоение банка (7-й раунд):** Если `ActivePlayers.Count == 2`, банк раунда ×2.
* **LastStrongestLinkName:** Запоминается для передачи в финальную дуэль.

### 2. PRE-GAME: Студийное вступление

Четыре фазы подготовки к эфиру:

| Кнопка | Аудиофайл | GameState | Суфлёр (HOST_MESSAGE) |
|--------|-----------|-----------|----------------------|
| OPENING | `main_theme_full.mp3` | IntroOpening | [ЛОГОТИП ИГРЫ] |
| INTRO 1 | `intro_track_1st.mp3` | IntroNarrative | Речь о шоу и 400к |
| INTRO 2 | `intro_track_2nd.mp3` | PlayerIntro | Представление участников |
| RULES | `intro_track_3rd.mp3` | RulesExplanation | Правила + цепочка + БАНК |
| READY | `playgame_with_general_bed.mp3` | RoundReady | «Начинаем с {Имя}!» |

* `PrepareNewRound()` вызывается автоматически (150 сек, первый по алфавиту).
* Суфлёр динамически подставляет имя первого игрока.

### 3. Сетевая синхронизация (Network)

* **TCP Server/Client:** Пульт = сервер (порт 8888), экраны = клиенты.
* **Broadcast Protocol:** `SET_STATE`, `UPDATE_BANK`, `UPDATE_TIMER`, `QUESTION`, `CURRENT_PLAYER`, `ELIMINATE`, `DUEL_UPDATE`, `WINNER`, `HOST_MESSAGE`, `HOST_MESSAGE|CLEAR`.
* **Суфлёр ведущего:** Полупрозрачный оверлей на HostScreen с крупным белым текстом.

### 4. Пульт Оператора (OperatorPanel)

* **PRE-GAME панель:** Кнопки OPENING → INTRO 1 → INTRO 2 → RULES → READY.
* **START O'CLOCK:** Ярко-зелёная кнопка запуска таймера раунда.
* **Быстрый ответ:** ВЕРНО, НЕВЕРНО, ПАС, БАНК — с визуальным эффектом нажатия (включая бот-режим).
* **VOTE START (45 сек):** Синхронизация с музыкой, 1 сек задержка для акцента.
* **REVEAL → VERDICT → ELIMINATE:** Полная цепочка послераундовой обработки.
* **NEXT ROUND:** Активна после ELIMINATE, блокируется после нажатия до READY.
* **CLOSE SESSION:** Возврат в предсессионный экран ввода участников.
* **Debug Panel:** Тестовые команды, бот-режим, быстрый переход к раундам.

### 5. StatusBar (Штурман оператора)

Трёхсегментная строка (50 px, Consolas):

| Блок | Заголовок | Назначение |
|------|-----------|------------|
| Левый | ВЫ СДЕЛАЛИ | Подтверждение последнего действия |
| Центр | СЕЙЧАС | Описание текущего процесса |
| Правый | ПОРА НАЖАТЬ | Прямое указание следующей кнопки (золотым) |

* Динамический цвет фона по состоянию.
* Подсказки на русском с английскими названиями кнопок.
* Интеграция с tie-detection (информация о ничье).

### 6. Система голосования и Tie-breaker

* **VOTE START:** Запуск 45-секундного таймера с аудио.
* **Auto-Stop:** При внесении всех голосов таймер останавливается.
* **REVEAL:** Вскрытие табличек с привязкой к аудио-таймингу.
* **IsTieDetected / GetTiedPlayerNames:** Определение ничьей при голосовании.
* **Сценарий А:** Сильное звено в числе ничьи — получает иммунитет, второй выбывает автоматически.
* **Сценарий Б:** Сильное звено решает между кандидатами. ComboBox фильтруется.
* **HOST_MESSAGE:** Автоматические суфлёрные тексты для каждого сценария.

### 7. Финальная дуэль (Head-to-Head)

* **5 пар вопросов:** Строгая очерёдность Игрок 1 → Игрок 2.
* **Досрочная победа:** Математическая невозможность отыграться → немедленный финиш.
* **Sudden Death:** При ничье 5:5 — до первого промаха в паре.
* **Match-ball суфлёр:** «Если ответите неверно — проиграете» и т.д.
* **HOST_MESSAGE:** Дуэльный сценарий, Sudden Death, победа.
* **WinnerPanel:** Финальный экран с именем и суммой.

### 8. Аналитика раунда (RoundStatsWindow)

* **Таблица:** Верные, неверные, пасы, банк, успешность, средний банк.
* **Сильное / Слабое звено:** Автоматическое определение.
* **Panic Banker:** Игрок с наименьшим средним банком.
* **Fix:** `savedRoundBank` передаётся до обнуления в движке.
* **Золотая рамка:** Подсветка Сильного звена при ничье.

### 9. Аудиодвижок (AudioManager / NAudio)

* **Раундовые треки:** Автовыбор по длительности.
* **Кроссфейд general_bed:** Fade-in за 3 сек до конца предыдущего трека.
* **Полная аудио-карта:**
  - PRE-GAME: `main_theme_full`, `intro_track_1st/2nd/3rd`, `playgame_with_general_bed`
  - Голосование: `new_voting_system v3START`, `revealing`, `before_walkshame`
  - Исключение: `updated_walk_of_shame_bed`
  - Финал: `final_round_prestart`, `final_round_start`, `sudden_death_bed`, `duel_winner`
* **Audio Priority:** Stop → Play для каждого нового трека.

### 10. Экраны ведущего

* **HostScreen (Classic):** Денежное дерево, таймер, банк, вопрос.
* **HostScreenModern (2020):** Горизонтальный «светофор».
* **Суфлёр:** `HostRulesOverlay` — полупрозрачный оверлей с крупным текстом.
* **Пульсация таймера:** Красный + анимация на последних 10 секундах.
* **Финал:** Кружочки результатов, имена финалистов.
